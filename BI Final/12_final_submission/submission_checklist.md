# Final Submission Checklist

## Purpose

I use this checklist to make sure the final project is complete, organized, and reproducible before submission.

## Required Files

- [ ] Final report in required format.
- [ ] Final presentation if required.
- [ ] Excel workbook.
- [ ] Power BI file if required.
- [ ] Python scripts.
- [ ] Cleaned data files.
- [ ] Merged pre-analysis or final analysis dataset.
- [ ] README.
- [ ] Data source documentation.
- [ ] Data dictionary.

## Recommended Supporting Files

- [ ] Validation checklist.
- [ ] Data quality report.
- [ ] Pipeline processing log.
- [ ] Artifact inventory.
- [ ] Export manifest.
- [ ] Project log.
- [ ] Assumptions log.
- [ ] Change log.
- [ ] Methodology notes.

## Organization Checks

- [ ] Folder structure is clean and numbered.
- [ ] Raw data is separate from cleaned data.
- [ ] Final submission files are separate from working outputs.
- [ ] No temporary files are included in the final package.
- [ ] File names follow the naming convention guide.

## Documentation Checks

- [ ] README explains the project purpose and workflow.
- [ ] Data source documentation explains why each dataset was included.
- [ ] Data dictionary includes source, unit, type, and role information.
- [ ] Limitations are documented without overstating findings.
- [ ] Reproducibility instructions are clear.

## Excel Checks

- [ ] Workbook has a README or documentation sheet.
- [ ] Raw import sheet is preserved.
- [ ] Integrated data table is formatted as an Excel Table.
- [ ] QA formulas are included.
- [ ] Chart-prep area is separated from raw import.
- [ ] Charts do not contain unsupported interpretation.

## Python Checks

- [ ] `requirements.txt` is present.
- [ ] Pipeline runs without errors.
- [ ] Scripts have clear first-person comments and docstrings.
- [ ] Cleaned files are exported.
- [ ] Merged files are exported.
- [ ] Validation reports are exported.
- [ ] Processing log is created.

## Power BI Checks

- [ ] Dataset imports successfully.
- [ ] Data types are correct.
- [ ] Date table is created or documented.
- [ ] Relationship between date table and fact table is correct.
- [ ] Dashboard pages are organized.
- [ ] Data quality page is included or planned.
- [ ] Visuals do not claim findings before analysis supports them.

## Report Checks

- [ ] Introduction is clear.
- [ ] Research question is stated.
- [ ] Dataset rationale is documented.
- [ ] Methodology is reproducible.
- [ ] Findings are supported by completed analysis.
- [ ] Recommendations follow from findings.
- [ ] Limitations are included.
- [ ] References are complete.

## Naming Checks

- [ ] Generated files use lowercase snake_case.
- [ ] Dates use `YYYY-MM-DD`.
- [ ] Draft and final versions are clearly labeled.
- [ ] Raw data file names remain traceable to sources.

## Reproducibility Checks

- [ ] Another person can follow the README and run the pipeline.
- [ ] The pipeline does not rely on hidden manual steps.
- [ ] Output folders are documented.
- [ ] Dependencies are listed.
- [ ] Data preparation decisions are recorded.

## Final Packaging Checks

- [ ] Final submission folder contains the required final files.
- [ ] Working files are included only if allowed or useful.
- [ ] No unsupported findings are present in setup documents.
- [ ] All placeholders in the final report are completed or removed before submission.
- [ ] I completed a final review of the package before submitting.
